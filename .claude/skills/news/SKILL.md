# /news — оркестратор новостей по 5 сайтам

Хаб-дирижёр. Реальную работу (сбор/рерайт/фото/перевод/QA/публикация) выполняет per-repo скилл `news` ВНУТРИ каждого клона. Хаб собирает превью в один дайджест, держит один гейт согласования и публикует во все.

**Команда:** `/news [страны] [N]` — страны опционально (по умолчанию все 5; можно `/news грузия албания`), N — сколько новостей на сайт (по умолчанию 3–5).

## Карта сайтов и клонов
| Страна | Клон | Язык/гео | Источники (в репо: `.claude/skills/news/SKILL.md`) |
|---|---|---|---|
| Грузия | `sites/gruzia-site` | ru/uk (СНГ) | грузинские (Agenda.ge/Civil.ge/аэропорты/Нацбанк), лари |
| Албания | `sites/albania-site` | en (Tier-1) | албанские (Tirana Times/TIA/punetejashtme.gov.al), лек |
| Черногория | `sites/montenegro-site` | en | gov.me/MUP/аэропорты Tivat/Podgorica, евро |
| Хорватия | `sites/croatia-site` | en | Total Croatia News/аэропорты/mup.gov.hr, евро |
| Македония | `sites/macedonia-site` | en | mfa.gov.mk/Skopje·Ohrid airport/Meta.mk, денар |

## Процедура (гейты не пропускать)

### 1. Подготовка
- Для каждого целевого сайта: `git -C sites/<repo> pull --ff-only`.
- Открой `sites/<repo>/.claude/skills/news/SKILL.md` — возьми оттуда актуальный список источников/гео/валюты (они уже локализованы под страну в Этапе 1).

### 2. Сбор + фильтр (по каждому сайту, можно параллельно агентами)
Для каждого сайта выполни ТОЛЬКО фазу сбора и фильтра из его репо-скилла:
- Собери кандидатов за окно (по умолчанию несколько дней) через WebSearch/WebFetch по курируемым источникам этого сайта.
- 3-ступенчатый фильтр: релевантность строго travel (визы/границы, авиа/аэропорты, цены/валюта, погода/ЧС, открытия-закрытия, **события для путешественника — фестивали, городские праздники, сезонные/культурные/гастрономические события, ярмарки, заметные концерты и спорт-события, привлекающие гостей**) → отсев политики/криминала/**рутинных** спорт-результатов/рекламы/кликбейта; дедуп (официальный > издание > агрегатор); анти-кликбейт.
- Отбор N на сайт **по факту** (ориентир 3–5, но без добивки ради числа; стоящего мало → меньше, пусто → пустой дайджест). Событие можно дать отдельной новостью или собрать афишей «что посмотреть в этом месяце».

### 3. Кросс-фактчек (ОБЯЗАТЕЛЬНО до дайджеста)
Прогони агента `cross-site-fact-checker` по каждому кандидату: подтвердить **дату/место/год/реальность** строго по источнику ТЕКУЩЕГО года. 🔴 Для ежегодных событий НИКОГДА не брать прошлогодние даты (реальный прокол был с фестивалем Батуми). Кандидат без подтверждения года/факта — выкинуть из дайджеста.

### 4. ОДИН дайджест владельцу — СТОП
Сведи всё в одну таблицу, сгруппированную по сайтам:
```
## 🇬🇪 Грузия
| # | тег | заголовок | суть (1–2 строки) | источник | фото | фактчек |
## 🇦🇱 Албания
...
```
Кратко отметь, что отсеяно. **Ничего не публикуем до явного «ок».** Владелец может вычеркнуть/поправить.

### 5. После «ок» — публикация (по каждому одобренному, в его репо)
В каждом `sites/<repo>` через его репо-скилл `news`:
- `npm run new -- news <slug>` → рерайт на языке сайта своими словами (1500–2000 знаков, 3–5 разделов, ≥4 внутр. ссылки) → фото ≥2 (cover ≤200КБ webp + ≥1 инлайн, CC/своё с атрибуцией) → переводы на ОСТАВЛЕННЫЕ языки сайта (Грузия en/ru/uk; Черногория/Хорватия en+ru; Албания/Македония только en — см. docs/STANDARDS.md; паритет: тот же slug/числа) → `npm run qa` = GO → `draft:false` → один коммит → `git push origin main`.
- Партнёрки только через `/go/{partner}`.

### 6. Завершение
- Подтверди владельцу, что и куда опубликовано (по сайтам, с ссылками/slug).
- Механика хранения уже в движке: лента 10 дней, главная 2 дня, ежедневный rebuild — отдельно делать ничего не нужно.

## Запреты
Не публиковать без фото/источника; не публиковать половину языковой тройки; не копировать чужой текст/заголовки дословно (§копирайт); не выдумывать факты; даты — только из источника текущего года.

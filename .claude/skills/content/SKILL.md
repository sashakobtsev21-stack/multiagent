# /content — оркестратор статей по календарям 5 сайтов

Хаб-дирижёр. Реальную работу выполняет per-repo скилл `add-content` ВНУТРИ каждого клона. Хаб выбирает темы из календаря каждого сайта, сводит превью и держит гейт согласования.

**Команда:** `/content [страны] [дата]` — страны опц. (по умолчанию все 5), дата опц. (по умолчанию сегодня; формат ДД.ММ или ДД.ММ.ГГГГ). Можно `/content албания 24.06` или `/content` (все на сегодня).

## Процедура

### 1. Выбор тем из календарей
- Для каждого целевого сайта: `git -C sites/<repo> pull --ff-only`, открой `sites/<repo>/KALENDAR.md`.
- Найди записи `[Статья]` на заданную дату (или ближайшие неопубликованные, если дата пустой день). Возьми из строки: заголовок, целевой запрос, slug, категорию, минимум фото, денежную связку, перелинковку.
- Если на дату стоит `📊 Замер GA/GSC` — напомни владельцу прислать данные аналитики (план корректируется по ним), статью в этот день не требуем.

### 2. Превью-план владельцу — СТОП
Сведи в одну таблицу: сайт | дата | заголовок | запрос | /slug/ | фото | 💰 | перелинк. **Не пишем до «ок».** (Это лёгкий гейт — подтвердить, что берём в работу именно эти темы.)

### 3. После «ок» — конвейер по каждой статье (в её репо)
В каждом `sites/<repo>` через его скилл `add-content`:
- Бриф `briefs/<slug>.md` (ключевой запрос, вопросы, ФАКТЫ из источников, фото-потребности) → `npm run new -- <тип> <slug> --title "…" [--category <cat>]` (генерит ОСТАВЛЕННЫЕ языки сайта + папку фото, draft:true; языки — docs/STANDARDS.md: Грузия en/ru/uk; Черногория/Хорватия en+ru; Албания/Македония только en).
- Черновик на ведущем языке (Грузия — ru; остальные — en): 1200–2000 слов, лид-абзац = прямой ответ, H2/H3 под вопросы, без FAQ, ≥2 внутр. ссылки + хаб, 1–3 AffiliateBox через `/go/`.
- Фото ОБЯЗАТЕЛЬНЫ по стандарту: статья ≥5 (cover+≥4), город ≥10, столица ≥15, маршрут — фото каждой остановки; webp ≤200КБ, CC/своё с атрибуцией, уникальные.
- Переводы на оставленные языки сайта (паритет: тот же slug, те же числа) → `fact-checker` (сверка с брифом, ловит выдуманное) → `content-editor` (DoD/копирайт/SEO/a11y/go).
- Гейты `npm run check · build · test · test:links · lint` зелёные → `draft:false` → коммит тройки → `git push origin main`.

### 4. Завершение
- Отчёт владельцу: что опубликовано по сайтам (slug/ссылки), что перелинковано.
- Синхронизируй статус: отметь опубликованные пункты в `sites/<repo>/KALENDAR.md`, обнови PROGRESS/HANDOFF, закоммить.

## Стартовый пакет (первые недели)
Для новых сайтов первые записи календаря — связный пакет: столица (статья `cities`, ≥15 фото) + 2–3 города (≥10) + 1 маршрут (фото каждой остановки) + P0-статьи деньги/доверие. Веди их так, чтобы статьи и будущие новости перелинковывались (тематические хабы).

## Правила
Факты только из источника/брифа (не выдумывать; нет данных → TODO + источник); живой человечный текст носителя языка, не ИИ-генерёнка; партнёрки только `/go/`; URL не менять (правка slug = 301 в `_redirects`).

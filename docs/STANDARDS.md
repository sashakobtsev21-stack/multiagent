# Стандарты контента (подтверждены 2026-06-22)

Действуют для всех 5 сайтов. Должны быть вписаны в `CLAUDE.md`/`CONTENT_GUIDE.md` каждого репо.

## Объём текста
- **Статья:** 1200–2000 слов (длинная форма; ранжируется в Google). Первый абзац — прямой ответ за 2–3 предложения; H2/H3 под вопросы; без «воды» и без FAQ-блока.
- **Новость:** 1500–2000 знаков (~250–350 слов), 3–5 разделов, ≥4 внутренние ссылки.

## Фото (минимум)
- **Статья:** ≥5 (cover + ≥4 инлайн/галерея).
- **Новость:** ≥2 (cover + ≥1 инлайн).
- **Маршрут:** cover + фото **каждой остановки**.
- **Город** (= статья `category=goroda`): ≥10.
- **Столица:** ≥15.
- Каждое фото уникально на материал; формат webp ≤200 КБ; только своё или CC/PD (Wikimedia/Openverse/Unsplash) с атрибуцией (автор + лицензия).

## Языки и гео (политика владельца 2026-06-22)
Языки по сайтам (остальные не использовать, см. память site-languages-policy):
- **Грузия** — en + ru + uk (ru/uk ведущие, СНГ; en — экспат/номад-трек).
- **Черногория** — en + ru (en ведущий, ru вторичный).
- **Хорватия** — en + ru (en ведущий, ru вторичный).
- **Албания** — только en.
- **Македония** — только en.

Паритет — только по оставленным языкам сайта (одинаковый slug, одинаковые числовые факты, взаимные hreflang + x-default). Новый контент делать только в оставленных языках.

## Тон и достоверность
- Живой, человечный текст носителя языка (uk — живая мова, не калька/суржик; en — как пишет носитель). НЕ ИИ-генерёнка.
- Факты только из источника/брифа — не выдумывать (цены/адреса/часы + «уточняйте» + дата; нет данных → TODO + источник).
- **Даты/время/числа — только из источника текущего года.** Для ежегодных событий никогда не копировать даты из прошлогодних статей (был реальный прокол с датами фестиваля).
- Партнёрские ссылки — только через `/go/{partner}` с `rel="sponsored nofollow noopener"`.

## Новости — механика (уже реализована в движке)
- Раздел хранит **10 дней** (`NEWS_SECTION_WINDOW=10`), блок на главной — **2 дня** (`NEWS_HOME_WINDOW=2`). Ежедневный rebuild подчищает ленту.
- `/news` — не слепой автопостинг: сбор кандидатов → 3-ступенчатый фильтр (релевантность travel / дедуп / анти-кликбейт) → превью владельцу (стоп до «ок») → рерайт своими словами + фото + переводы → QA-гейт → публикация.

# Стандарты контента (подтверждены 2026-06-22)

Действуют для всех 5 сайтов. Должны быть вписаны в `CLAUDE.md`/`CONTENT_GUIDE.md` каждого репо.

> **ТРИ ЖЁСТКИХ ГЕЙТА КАЧЕСТВА перед `draft:false` (равны по силе `npm run qa`): ① объём · ② человечность (анти-ИИ) · ③ фото.** Любой не пройден — материал **НЕ публикуется**, переписывается. Это гейты, а не пожелания.

## Объём текста
- **Статья:** 1200–2000 слов (длинная форма; ранжируется в Google). Первый абзац — прямой ответ за 2–3 предложения; H2/H3 под вопросы; без «воды» и без FAQ-блока.
- **Новость:** 1500–2000 знаков (~250–350 слов), 3–5 разделов, ≥4 внутренние ссылки.

## Фото (минимум)
- **Статья:** ≥5 (cover + ≥4 инлайн/галерея).
- **Новость:** ≥2 (cover + ≥1 инлайн).
- **Маршрут:** cover + фото **каждой остановки**.
- **Город** (= статья `category=goroda`): ≥10.
- **Столица:** ≥15.
- Каждое фото уникально на материал; формат webp ≤200 КБ; только своё или CC/PD (Wikimedia/Openverse/Unsplash) с атрибуцией (автор + лицензия).

## Языки и гео (политика владельца 2026-06-22)
Языки по сайтам (остальные не использовать, см. память site-languages-policy):
- **Грузия** — en + ru + uk (ru/uk ведущие, СНГ; en — экспат/номад-трек).
- **Черногория** — en + ru (en ведущий, ru вторичный).
- **Хорватия** — en + ru (en ведущий, ru вторичный).
- **Албания** — только en.
- **Македония** — только en.

Паритет — только по оставленным языкам сайта (одинаковый slug, одинаковые числовые факты, взаимные hreflang + x-default). Новый контент делать только в оставленных языках.

## ② Человечность (анти-ИИ) — ЖЁСТКИЙ ГЕЙТ
Текст обязан звучать как написанный носителем-человеком, который разбирается в теме / был на месте. **Перед `draft:false` — прогон через `content-editor` с задачей вычистить ИИ-тон; не прошёл — переписать, не публиковать** (равный по силе гейт с qa/объёмом/фото). Любой пункт ниже провален → переписать:
1. **Лид — прямой ответ** (2–3 предложения). НЕ «В этой статье мы рассмотрим / Добро пожаловать / Представьте себе…».
2. **Ноль ИИ-штампов и канцелярита.** ru: «является / представляет собой», «стоит отметить / важно понимать», «в заключение / таким образом / более того», «играет ключевую роль», «незабываемые впечатления / не оставит равнодушным / жемчужина / райский уголок / поражает воображение / богатая история и культура». en: «nestled / boasts / a hidden gem / breathtaking / must-visit / whether you're a… or a… / something for everyone / when it comes to / in conclusion».
3. **Конкретика, не вода:** каждый абзац несёт факт/цифру/имя/деталь из источника; никаких общих фраз, применимых к любой стране («здесь много интересного», «на любой вкус»).
4. **Живой ритм:** разная длина предложений, НЕ шаблон «Х — это… Здесь можно…» в каждом абзаце; без бесконечных параллельных перечислений.
5. **Голос практика:** «как доехать», реальные цены/часы (из источника + «уточняйте»), локальные детали; пишет человек, не энциклопедия. Уместный лёгкий разговорный тон — да; вода и канцелярит — нет.
6. **Язык носителя:** en — как пишет нейтив; ru — живой русский (не калька с en); uk — жива мова (не суржик/калька). Перевод без адаптации НЕ публикуется.
7. **Объём набирается фактами, а не водой:** не растягивать ради знаков (это пересекается с гейтом ① — длина должна быть наполнена).

## Тон и достоверность
- Живой, человечный текст носителя языка (uk — живая мова, не калька/суржик; en — как пишет носитель). НЕ ИИ-генерёнка (детальный гейт — раздел «② Человечность» выше).
- Факты только из источника/брифа — не выдумывать (цены/адреса/часы + «уточняйте» + дата; нет данных → TODO + источник).
- **Даты/время/числа — только из источника текущего года.** Для ежегодных событий никогда не копировать даты из прошлогодних статей (был реальный прокол с датами фестиваля).
- Партнёрские ссылки — только через `/go/{partner}` с `rel="sponsored nofollow noopener"`.

## Перелинковка (внутренняя + кросс-сайтовая)
**Внутренняя (within site) — гейт `check-interlinks`:**
- Каждая статья — **≥2 ссылки на связанные статьи**; новость — **≥4**. Хаб раздела автоматически перечисляет свои статьи (ручных списков нет).
- **Связь в обе стороны:** новую статью линковать И из родственных материалов (входящая ссылка), а не только наружу. **Не плодить «сирот»** — вечнозелёную статью без единой входящей ред. ссылки вплести в связанные (новости-исключение: ротируются за 10 дней, входящие не обязательны).
- Все внутренние ссылки — относительные `/{категория}/{slug}/` со слешем; для ru/uk — с префиксом `/ru/`(`/uk/`). 0 битых (`test:links` = GO). Не ссылаться на draft/несуществующие.

**Кросс-сайтовая (между 5 сайтами сети) — ОЧЕНЬ аккуратно:**
- **Только контекстные** ссылки на реально стыкующихся темах (приграничный транспорт «страна А→Б», прибрежные/мульти-страновые маршруты, общая гео типа озера Охрид). **1 ссылка на статью, не футер-ферма** (массовый шаблонный кросс-линк Google трактует как искусственную сеть/PBN → риск пессимизации).
- **Цель обязана существовать и резолвиться** (проверить `curl` → 200 на проде сестринского сайта) — не ссылаться на ещё-не-созданные страницы.
- Писать обычной markdown-ссылкой на полный `https://`-URL сестринского домена с языковым префиксом (`/ru/`): движок (`rehypeExternalLinks`) **авто-навешивает `rel="nofollow noopener noreferrer"`** → нулевой PBN-риск, реферальный трафик сохраняется.
- Каденс — по мере выхода органичных приграничных статей; реципрокность желательна, но без натяжки.

## Новости — механика (уже реализована в движке)
- Раздел хранит **10 дней** (`NEWS_SECTION_WINDOW=10`), блок на главной — **2 дня** (`NEWS_HOME_WINDOW=2`). Ежедневный rebuild подчищает ленту.
- `/news` — не слепой автопостинг: сбор кандидатов → 3-ступенчатый фильтр (релевантность travel / дедуп / анти-кликбейт) → превью владельцу (стоп до «ок») → рерайт своими словами + фото + переводы → QA-гейт → публикация.

## Находки

### [Medium 🛡] Нет Consent Mode v2 и cookie-баннера при GA4 на всех страницах
**Доказательство:** `src/layouts/BaseLayout.astro:142-148` — GA4 (`G-PV88LQSD3W`) грузится безусловно в `<head>`; нет ни одного `gtag('consent','default',…)`, нет баннера согласия (grep по `consent` в components/layouts — 0 в контексте аналитики). Live-CSP это подтверждает (gtag разрешён).
**Влияние:** для EU-трафика (GSC: Нидерланды 523 показа — 2-е гео Грузии, плюс DE) сбор аналитики без согласия = риск GDPR/ePrivacy; для US/CIS — нейтрально. Consent Mode v2 также влияет на полноту данных Google Ads/моделирование конверсий, если позже появится `/reklama/`.
**Фикс:** добавить `gtag('consent','default',{ad_storage:'denied',analytics_storage:'denied',…})` ДО `config`, лёгкий баннер (vanilla, без сторонних CMP — правило §8 «одна аналитика»), при согласии `gtag('consent','update',…)`. Пересчитать sha256 инлайна в `_headers`. Тиражировать на все 5 форков (память `shared-engine-port-fixes-to-all-forks`).
**Усилие:** M · **Вердикт:** реальный, единственный системный пробел; не блокирует, но для EU-доли — приоритет №1 по compliance.

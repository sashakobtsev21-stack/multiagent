# Аудит агентов-писателей сети + фикс резолва (2026-07-01)

Задача 9 ROADMAP. Аудит и улучшение всех агентов, пишущих/проверяющих контент, по 8 репо; выравнивание с редколлегией `/work`; разбор «почему `/work` не задействует специализированных».

## 1. Корневая причина «почему дженерики» (РЕШЕНО)
`/work` работает с **хаб-уровня** (`multiagent/`). Специализированные агенты (`seo-content-writer`, `content-editor`, `fact-checker`, `research-collector`, переводчики и т.д.) лежат в **`sites/<repo>/.claude/agents/`** — репо-локально. **Из хаба они НЕ резолвятся как `subagent_type`**: оркестратору видны только хаб-`.claude/agents/` (был один — `cross-site-fact-checker`) + глобальные типы (`general-purpose`, `Explore`, …). Поэтому ранний прогон 01.07 брал дженерик-агентов с инлайн-промптами — не потому что специалисты плохие (они хорошие), а потому что **их нечем было вызвать по имени**.

## 2. Фикс — хаб-уровневая РЕДКОЛЛЕГИЯ (network-generic, резолвится из /work)
Заведены в `multiagent/.claude/agents/` (читают контекст целевого репо в рантайме — `sites/<repo>/CLAUDE.md` + `docs/STANDARDS.md`; параметр — репо + slug(и)):

| Агент | Модель | Роль (слой /work) |
|---|---|---|
| `content-writer` | Opus | производство: ресёрч-бриф→текст→фото→перевод, 1 на клон |
| `content-fact-checker` | Sonnet | слой 1: правдивость (факт↔источник + web-спот) |
| `content-reviewer` | Sonnet | слой 1: человечность + интересность + грамматика (② STANDARDS) |
| `language-reviewer` | Sonnet | слой 1: семантика+естественность ru/uk (ловит «платки»=tolls) |
| `editor-in-chief` | Opus | слой 2: финальное GO/NO-GO по сути, 1 на сайт |
| `cross-site-fact-checker` | (как было) | новости (дата/место/год/реальность) |

`/work` SKILL обновлён: слои зовут этих хаб-агентов поимённо; добавлена заметка про резолв (не поднимать дженерик под роль, для которой есть хаб-специалист). Репо-локальные агенты остаются для работы ВНУТРИ репо (`/add-content`).

## 3. Canonical-per-intent — форвард-процесс (не ретро-чистка)

Решение владельца «одна интент-страница» = **форвард-дисциплина**, а не разовая зачистка (сеть уже чиста по гейту). Процесс:
- **STANDARDS §④ «Уникальность намерения»** — жёсткий гейт публикации: один поисковый интент = одна страница; вариации/сателлиты — folds внутри или отдельная страница с `canonical` на основную.
- **Один город = одна статья** `category=cities`/`goroda` (не плодить «город» + «достопримечательность» под одним именем — класс C3, исторический дубль `borjomi`/Грузия).
- **Транспорт:** одна страница на направление; `X-to-Y` и `Y-to-X` — НЕ две страницы (гейт `b` ловит reverse-пары).
- **Itinerary/маршруты:** не плодить «N-day» вариации под один запрос (исторически худшая каннибализация — Македония itinerary 5→1-2); развести по интенту (planning vs logistics, как montenegro 2.1) или свести в pillar.
- При сомнении «дубль или нет» — решает интент поиска, не лексика заголовка (гейт лексику уже проверил).

## 4. Дедуп будущих `○`-слотов календарей (упреждающе, до написания)

Календари 4 EN-сайтов до 31.07 несут `○`-темы (croatia ~75, montenegro/albania/macedonia 3/будень). **До написания каждой `○`-темы** — прогон по чек-листу анти-каннибализации:

1. **Поиск интент-двойника:** есть ли уже опубликованная страница под тот же поисковый интент (не только похожий заголовок)? Если да — это `folds`/раздел существующей или сателлит с `canonical`, НЕ новая страница.
2. **Транспорт-темы:** для нового `X-to-Y` проверить отсутствие `Y-to-X` (иначе гейт `b` упадёт при публикации).
3. **Город-темы:** одна статья на город (`category=cities`), не дубль «город»+«достопримечательность».
4. **Itinerary/«N-day»-темы:** не добавлять вариацию длительности под существующий маршрут без явного развода интента.
5. **Заголовок:** держать token-Jaccard с существующими <0.85 (иначе гейт `c` упадёт) — но это вторично; первично — разный интент.
6. При публикации статьи N — сразу ≥1 входящая ссылка из тематически близкой (анти-сирота, гейт `check-interlinks` MIN=2) + ≥2 исходящих контекстных.

**Метрика прогресса кластера = позиция/показы по GSC, НЕ +N страниц** (решение владельца 1I). Если кластер уже ранжируется — углублять/перелинковывать существующее, а не плодить вариации (риск самоканнибализации). Сверка GSC — Пн 06.07.

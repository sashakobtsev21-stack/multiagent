## Сетевое здоровье — едино по 5 аудитам + обеим критикам

**Техническая платформа подтверждена по конфигам и git (не выдумана):**

- SEO-мета: title≤60 / desc≤155 **в символах** (не байтах) — 0 нарушений на 4 EN-сайтах; у Грузии 1 desc=156 (новость, косметика). Байтовый счёт даёт ложные тревоги на кириллице (память `seo-meta-length-chars-not-bytes`).
- Безопасность: строгий CSP без `unsafe-inline` в script-src, HSTS preload, XFO DENY, Permissions-Policy, anti-open-redirect в `/go/` (https-only target, относительный фолбэк). Зелено везде.
- Монетизация-дисциплина: 0 прямых партнёрских URL в телах статей во всех 5; `/go/{partner}` + `rel="sponsored nofollow noopener"`; SubID-атрибуция живая (проверено curl: `/go/trip-hotels?c=… → trip_sub1=…`).
- Фото-гейт ≥5: 0 статей под порогом во всей сети (независимо пересчитано) → **P1-4 «фото-дефицит» закрыт**.
- Гейт ④ `check-dedup`: live в `npm test`/`qa`, бейслайны пусты во всех 5 → анти-каннибализация активна «по-настоящему», без grandfathered-исключений.
- Форк-чистота: 0 реальных грузинских хвостов в боевом `src`/`scripts` 4 форков (остатки — в `.claude/agents/*` и осознанные комментарии-провенанс).

**Critical — нет ни на одном сайте. High — фактически нет** (см. ниже: единственный сетевой High-кандидат EU-compliance Грузии и единый сетевой долг EKTA; оба не блокируют публикацию).

**Главные риски достоверности — не в технике, а в силе формулировок про спрос** (общий вывод обеих критик): тезисы «insurance = деньги №1 / быстрый выигрыш», «nightlife/релокация — спрос» опираются на 1 неделю GSC-песочницы (60–100 показов, позиции 64–94, ~0 кликов) и отраслевую логику. Это обоснованные **гипотезы/направления роста до замера 06.07**, а не доказанный доход «сейчас». Сами per-repo аудиты и baseline это в основном оговаривают; в сводке тон выровнен на «потенциал».

## Сквозные находки сети (после оппонирования)

1. **EKTA — единый сетевой долг, severity выровнена в Low/owner-action.** `partners.json` всех 5: `ekta = https://ektatraveling.com/`, `allowSubId:false` — голый URL без атрибуции (факт ПОДТВЕРЖДЁН). НО live-использование во всём контенте сети — **ровно 1 файл**: `macedonia-site/.../driving-in-north-macedonia.md:22` (`partner:'ekta'`); в gruzia/albania/montenegro/croatia EKTA **не подключён ни к одной странице** (перепроверено `grep partner:'ekta'` = 0). Аудиты оценили один и тот же факт рассогласованно: **Albania High, Macedonia High, Montenegro Medium, Croatia Low, Georgia Low**. Обе критики сходятся: High у Albania **переоценён** (0 использований → терять нечего), наиболее защищён Macedonia (единственная живая, но «1 клик владельца»). Корректная единая шкала по сети = **Low / owner-action** (код исправен, действие внешнее — рабочая tpx.gr-ссылка из кабинета Travelpayouts). Важно: insurance-кластер EKTA **не касается** — все insurance-страницы монетизируются через `safetywing` (см. п.2).

2. **Insurance-кластер монетизируется через SafetyWing — но subID не идёт и у него (НУЖНО ДОКАЗАТЕЛЬСТВО).** Все три insurance-страницы (`albania-travel-insurance`, `croatia-travel-insurance`, `north-macedonia-travel-insurance`) + montenegro несут `safetywing`. Критика «деньги» вскрыла асимметрию: SafetyWing тоже `allowSubId:false` (статический `referenceID`, без per-slug subID), и ни один аудит это не отметил. Различие реально (referenceID атрибутирует на уровне аккаунта, голый EKTA-URL — ничего), но «страховой кластер монетизируется корректно» — недоказуемо в READ-only (конверсия в кабинете Travelpayouts/Ambassador не верифицировалась, только наличие ID в URL). Не блокер; пометка для владельца при подключении рекламы/расширении.

3. **Montenegro «болванки» discovercars/getyourguide/booking — мёртвые ключи, не потеря дохода.** `partners.json` Черногории: три прямых URL без атрибуции (`allowSubId:false`) — ПОДТВЕРЖДЕНО (VERIFY 4a + curl: `/go/discovercars?c=test → 302 голый домен`). НО `grep partner:` по контенту → **0 использований**; EN-статьи берут `trip-carhire`/`safetywing` с рабочим SubID. Сам montenegro-аудит поставил Medium с оговоркой «живой потери нет» — это **обоснованно** (housekeeping/owner-action, не открытый денежный долг P1, как звучало в тоне VERIFY).

4. **Грузия — единственный сайт без Consent Mode v2; противоречит `shared-engine-port-fixes-to-all-forks`.** Перепроверено: gruzia — `CookieConsent.astro`=0, consent-js=0; **все 4 форка** несут `CookieConsent.astro` + ga-init/consent-banner. Парадокс: фикс прошёл «forks-first», а на origin-движок (Грузия) не портирован — прямое нарушение memory-правила, которое **ни один per-repo аудит как кросс-сайт-противоречие не отметил** (отметила критика «доказательства» B2). Усиливает gruzia-Medium: для EU-доли Грузии (NL — 2-е гео, 523 показа) GA4 грузится безусловно без согласия = единственный реальный GDPR-пробел сети.

5. **Albania future-date `publishedAt: 2026-07-08` на живом маршруте — ПОДТВЕРЖДЕНО (дисциплина, не деньги).** `src/content/routes/en/albania-road-trip-itinerary.md`: `publishedAt/updatedAt: 2026-07-08`, `draft:false`, live HTTP 200; реальная публикация (git) — 2026-06-22. Единственный такой случай в сети (GE/ME/HR/MK чисто). Корень — `publishedAt` забит будущим слотом календаря вместо даты факта (класс C8). Обе критики: верно, но к монетизации нейтрально.

6. **Тонкие money/evergreen-страницы (<1200 слов) — вердикт держится, конкретные числа в аудитах «плавают».** Независимый пересчёт тела (strip frontmatter+теги) показал систематически меньшие числа, чем в аудитах, но порог 1200 не достигнут — то есть **вердикт «тонкая» сохраняется, цифрам аудитов как точным доверять нельзя** (±90–230 слов из-за методики подсчёта figure-подписей):

Имена слагов в albania-аудите местами неточны (реальные — `do-you-need-insurance.md`, `digital-nomad-visa.md`).
   - Croatia `croatia-digital-nomad-visa` (аудит 958) → **~876**.

   - Gruzia `ananuri`/`cherepashje-ozero` — десятки–сотни слов, но это фокус-страницы остановок маршрутов, тон человечный (не ИИ).

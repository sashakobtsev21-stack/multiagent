## ЗАДАЧИ A–F

### A. Полный технический аудит (скилл /full-audit)
Запусти скилл **/full-audit** (`.claude/skills/full-audit/SKILL.md`). Он прогоняет `npm run qa` и `npm run qa:browser` с выводом в `_scratch/audit/` (`qa.txt`, `qa-browser.txt`, скриншоты шаблонов мобайл+десктоп). Покрой все направления и зафиксируй каждую находку с severity (critical / high / medium / low):
- **Код:** чистая сборка без ошибок, типы (`astro check`), мёртвый код, битые импорты, конфиги (`astro.config.mjs`, `wrangler.jsonc`, `worker/index.ts` — у этого репо они есть, проверь что рабочие).
- **Безопасность:** CSP и `public/_headers`; open-redirect в `/go/{partner}` — редирект должен идти ТОЛЬКО по whitelisted-ключам из `src/data/partners.json` (это keyed-карта, произвольные URL запрещены), при неизвестном ключе — `fallback`; утечка секретов/токенов в репо и истории; зависимости и известные CVE.
- **Производительность:** LCP/CLS, JS-бюджет, размеры изображений (webp ≤200 КБ), отсутствие тяжёлых блокирующих ресурсов; lazy-карты Leaflet и click-to-load виджеты не должны грузить JS до взаимодействия.
- **Тех-SEO:** `schema.org` (разметка статей), `hreflang` + `x-default`, `canonical`, `sitemap` (i18n), `robots.txt`. Проверь, что НЕТ грузинских доменов/путей в canonical/hreflang/sitemap/редиректах.
- **Доступность (WCAG):** alt у изображений, контраст, фокус, семантика, навигация с клавиатуры.
- **UI/UX на ДЕСКТОПЕ и МОБАЙЛЕ:** пройди каждую кнопку / клик / скролл / ссылку — всё ли работает; адаптив (нет горизонтального скролла на 320–1280, тапабельные зоны, читаемость). Проверь шапку, футер, карточки, меню языков, ссылки на статью, `/go/`-редиректы.

Прогони реальные команды и приложи результаты к находкам:
```
npm run qa
npm run qa:browser
```
(`qa` = чистая сборка + статический аудит dist; `qa:browser` = mobile-Lighthouse + responsive 320–1280.) Если команда падает — это находка, зафиксируй причину и severity.

Результат A → собери в файл **AUDIT-2026-06-22.md** (создай в корне репо): список находок с severity, путями к файлам и кратким «как чинить».

## СТАНДАРТЫ (вписать в CLAUDE.md и CONTENT_GUIDE.md)

**Объём:**
- Статья — **1200–2000 слов** (длинная форма). Первый абзац — прямой ответ за 2–3 предложения; H2/H3 под вопросы; без воды и без FAQ-блока.
- Новость — **1500–2000 знаков** (~250–350 слов), 3–5 разделов, **≥4 внутренние ссылки**.

**Фото (минимум):**
- Статья — **≥5** (cover + ≥4 инлайн/галерея).
- Новость — **≥2** (cover + ≥1 инлайн).
- Маршрут — cover + фото **каждой остановки**.
- Город (= статья `category=goroda`) — **≥10**.
- Столица (Загреб) — **≥15**.
- Каждое фото уникально на материал; формат **webp ≤200 КБ**; только своё или CC/PD (Wikimedia / Openverse / Unsplash) с атрибуцией (автор + лицензия).

**Контент-модель:**
- **Город = статья с `category=goroda`.** Коллекция `cities` НЕ используется.

**Тон и достоверность:**
- Живой, человечный текст носителя языка (en — как пишет нейтив; uk — живая мова, не калька/суржик). НЕ ИИ-генерёнка.
- Факты только из источника/брифа — не выдумывать. Нет данных → `TODO` + источник.
- Даты/время/числа — только из источника **текущего года**; для ежегодных событий никогда не копировать прошлогодние даты.

**Монетизация:**
- Партнёрские ссылки — только через `/go/{partner}` с `rel="sponsored nofollow noopener"`. Прямые URL партнёрок в контенте запрещены.

## A) Полный технический аудит (`/full-audit` или эквивалент)
Прогони `npm run qa` и `npm run qa:browser` и проведи аудит по всем направлениям. Каждую находку фиксируй с **severity** (critical / high / medium / low) и конкретным путём к файлу:

- **Безопасность:** CSP и `_headers` (если есть); open-redirect в обработчике `/go/` (партнёрские редиректы — только на белый список доменов); утечки секретов/токенов в репо; зависимости и известные CVE (`npm audit`).

- **Доступность (a11y / WCAG):** alt у изображений, контраст, фокус, навигация с клавиатуры, landmark-роли.
- **UI/UX на ДЕСКТОПЕ и МОБАЙЛЕ:** пройди по реальным страницам — каждая кнопка, клик, скролл, ссылка, переключатель языка работают; меню и карта (Leaflet) живые; адаптив без поломок и горизонтального скролла. Проверь оба сид-материала.
